####抽象工厂模式(AbstractFactory Pattern)
           
######抽象工厂模式围绕超级工厂创建其他工厂。属于创建型模式。

* 工厂模式产生的目的
######提供一个创建一系列相关或相互依赖对象的借口，无需指定具体类
* 工厂模式解决的问题
######接口选择问题；系统具备多个产品，但只消费其中一个
* 关键代码
######一个工厂内聚合多个相同类别产品
* 优点
######一个产品族中多个对象一同工作时，可保证客户端始终只使用同一产品族对象
* 缺点
######产品族扩展困难，增加某一系列某一产品，需要同时改动工厂及实现类，破坏封装性
* 使用场景
######根据不同环境提供相同功能
